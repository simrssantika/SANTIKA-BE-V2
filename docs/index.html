SANTIKA SIMRS

Sistem Informasi Manajemen Rumah Sakit berbasis modular monolith, dibangun dengan Spring Boot 4 + Kotlin. Menggunakan dual ORM (JPA untuk CRUD ringan, jOOQ untuk query kompleks) dengan DragonFly sebagai cache dan rate limiter.

Setup Database

1

Buat database PostgreSQL

CREATE DATABASE simrs;
2

Konfigurasi koneksi

Default sudah terset di application.yaml. Override lewat env variable:

ConfigDefault
DB_URLjdbc:postgresql://localhost:5432/simrs
DB_USERNAMEpostgres
DB_PASSWORDroot
3

Migrasi otomatis via Liquibase

Liquibase berjalan saat aplikasi start. Schema yang dikelola:

  • MASTER — data master (poli, diagnosa, wilayah, penjamin)
  • SHARED — lintas modul (user, role, permission, audit, file)
  • KEPEGAWAIAN — data kepegawaian
# Tanpa seed data (default)
LIQUIBASE_SEED=NON

# Dengan seed data development
LIQUIBASE_SEED=DEV

Setup DragonFly

DragonFly adalah pengganti Redis yang sepenuhnya kompatibel dengan protokol Redis dan lebih efisien dalam penggunaan memori.

Cara Install & Jalankan

Native (Linux / Mac)

dragonfly --requirepass root --maxmemory 512mb --maxmemory-policy allkeys-lru

Docker

docker run -d \
  --name simrs-dragonfly \
  -p 6379:6379 \
  docker.dragonflydb.io/dragonflydb/dragonfly \
  --requirepass root \
  --maxmemory 512mb \
  --maxmemory-policy allkeys-lru

Konfigurasi Koneksi

VariableDefault
DRAGONFLY_HOSTlocalhost
DRAGONFLY_PORT6379
DRAGONFLY_PASSWORDroot

Fungsi DragonFly di Aplikasi Ini

FungsiDetail
Cache authuserAuthorities dan userPrincipalDetail — TTL 5 menit
Rate limitingCounter per IP + method + endpoint — window 1 menit
File upload lockTracking upload aktif — mencegah cleanup berjalan saat ada upload

Menjalankan Aplikasi

1

Clone & masuk direktori

git clone <repo-url>
cd simrs
2

Generate jOOQ classes

jOOQ codegen dijalankan dari Liquibase changelog — tidak perlu koneksi DB live.

./gradlew generateJooq
3

Jalankan aplikasi

# Development
./gradlew bootRun

# Production — build JAR dulu
./gradlew bootJar
java -jar build/libs/simrs-0.0.1-SNAPSHOT.jar
✓ Aplikasi berjalan di port 9002 — http://localhost:9002

Rate Limiting

Diterapkan otomatis ke semua endpoint /api/** per kombinasi IP + HTTP method + endpoint. Disimpan di DragonFly dengan window 1 menit.

Grup EndpointLimitAlasan
/api/v1/auth/**5 request / menitCegah brute force login
Endpoint lainnya60 request / menitLimit umum per IP

Jika limit terlampaui, client menerima:

HTTP 429 Too Many Requests
Retry-After: 45

{
  "status": 429,
  "message": "Terlalu banyak permintaan. Coba lagi dalam 45 detik"
}
ℹ Jika DragonFly tidak tersedia, rate limiter fail-open — semua request tetap diizinkan agar aplikasi tidak crash karena Redis down.

Circuit Breaker

Hanya aktif untuk koneksi keluar (external service). Database internal tidak dipasang CB — baca alasan desain ini di sesi diskusi.

Instance yang Tersedia

InstanceDipakai untukPerilaku jika mati
dragonflyKoneksi ke DragonFly (rate limit, cache)Fail-open — request tetap jalan
externalPDF service / Go service / HTTP eksternalFallback → exception dengan pesan jelas

Cara Pakai di Service

@CircuitBreaker(name = "external", fallbackMethod = "generatePdfFallback")
@Retry(name = "external")
fun generatePdf(data: PdfRequest): ByteArray {
    // panggil Go service / PDF service
}

private fun generatePdfFallback(data: PdfRequest, ex: Exception): ByteArray {
    throw ServiceUnavailableException("PDF service sedang tidak tersedia")
}

CB untuk DB (jika DB Remote)

Jika database dipindah ke jaringan berbeda, aktifkan config yang sudah disiapkan di application.yaml:

# Uncomment blok ini di application.yaml
instances:
  database:
    base-config: default
    sliding-window-size: 10
    failure-rate-threshold: 50
    wait-duration-in-open-state: 15s
    ...
// Tambahkan anotasi di service layer
@CircuitBreaker(name = "database", fallbackMethod = "fallback")
fun queryDatabase(): T { ... }

Struktur Modul

src/main/kotlin/com/santika/simrs/
├── global/ # Cross-cutting: cache, rate limit, security, JWT, exception
│ ├── cache/ # RedisCacheManager (DragonFly)
│ ├── config/ # JooqConfig, WebConfig, SchedulingConfig
│ ├── exception/ # GlobalException handler
│ ├── jwt/ # JwtTokenProvider, JwtAuthFilter
│ ├── ratelimit/ # RateLimitInterceptor (DragonFly)
│ ├── response/ # BaseResponse, PageResponse, SliceResponse, OptionsRes
│ └── security/ # SecurityConfiguration, UserContext
│
├── module/
│ ├── kepegawaian/ # Modul HR — pegawai, dokter, departemen, dll
│ └── master/ # Data master — wilayah, medis, penjamin
│
└── shared/ # Fitur lintas modul
├── auth/ # AuthService, AuthController (login, refresh)
├── file/ # File system (temp, storage, lock, cleanup)
├── role/ # Role & Permission management
└── user/ # User management
ℹ Dual ORM: Gunakan JPA untuk operasi CRUD sederhana (save, findById, delete). Gunakan jOOQ untuk query dengan join banyak tabel, filter kompleks, atau yang butuh performa tinggi.
